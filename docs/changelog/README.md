## 版本命名规则

采用语义化版本号：`主版本号.次版本号.修订号`

- **主版本号（Major）：** 重大架构变更或不兼容的API修改
- **次版本号（Minor）：** 新增功能，向后兼容
- **修订号（Patch）：** Bug修复、文档更新，向后兼容
