## 文档命名规范

每次改进文档使用以下命名格式：
```
v{版本号}-{日期}.md
```

例如：`v1.2.0-2026-03-29.md`

## 文档内容结构

每个改进文档应包含：

1. **改进版本和日期**
2. **测试结果概览**（如有）
3. **主要改进内容**
   - 问题描述
   - 改进措施
   - 预期效果
4. **文件改动清单**
5. **改进效果预测**（如有）
6. **下一步建议**
7. **总结**
