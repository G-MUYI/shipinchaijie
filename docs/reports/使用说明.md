# 报告管理系统使用说明

## 目录结构

```
视频拆解skills/
├── docs/
│   └── reports/                    # 统一的报告管理目录
│       ├── README.md               # 报告索引（包含所有报告的目录）
│       └── project-review-2026-03-29.md  # 具体报告文件
├── shipinchaijie/
│   └── docs/
│       └── improvements/           # 技能改进历史（技能内部文档）
│           ├── README.md
│           ├── v1.2.0-2026-03-29.md
│           └── v2.1.0-2026-03-29.md
└── README.md                       # 项目主文档（已添加报告目录链接）
```

## 报告类型

### 1. 项目审查报告
- **位置：** `docs/reports/project-review-YYYY-MM-DD.md`
- **用途：** 全面评估项目质量（架构、功能、文档、测试等）
- **频率：** 重大版本发布前，或定期审查

### 2. 评估测试报告
- **位置：** `docs/reports/eval-report-iteration-N-YYYY-MM-DD.md`
- **用途：** 验证技能在测试用例上的表现
- **频率：** 每次技能改进后

### 3. 性能测试报告
- **位置：** `docs/reports/performance-report-YYYY-MM-DD.md`
- **用途：** 评估处理时间、资源消耗等
- **频率：** 性能优化前后

### 4. 描述优化报告
- **位置：** `docs/reports/description-optimization-YYYY-MM-DD.md`
- **用途：** 优化技能描述以提升触发准确率
- **频率：** 发现触发问题时

## 如何添加新报告

### 步骤 1：创建报告文件

```bash
# 在 docs/reports/ 目录创建新报告
cd docs/reports/
touch eval-report-iteration-1-2026-03-30.md
```

### 步骤 2：编写报告内容

按照报告类型的标准格式编写内容。

### 步骤 3：更新索引

编辑 `docs/reports/README.md`，在对应的表格中添加新报告的索引条目：

```markdown
| 2026-03-30 | [评估测试报告 - 迭代1](eval-report-iteration-1-2026-03-30.md) | 5 | 80% | 首次评估测试 |
```

### 步骤 4：提交到 Git

```bash
git add docs/reports/
git commit -m "docs: 添加评估测试报告 - 迭代1"
```

## 与改进历史的区别

| 维度 | 报告 (docs/reports/) | 改进历史 (shipinchaijie/docs/improvements/) |
|------|---------------------|-------------------------------------------|
| **范围** | 整个项目 | 单个技能 (shipinchaijie) |
| **内容** | 审查、评估、性能测试等 | 版本改进、功能变更 |
| **视角** | 外部审查、质量评估 | 内部开发、迭代记录 |
| **受众** | 项目维护者、审查者 | 技能使用者、开发者 |

## 归档策略

当报告数量超过 20 个时，建议按年份归档：

```
docs/reports/
├── README.md
├── 2026/
│   ├── project-review-2026-03-29.md
│   └── eval-report-iteration-1-2026-03-30.md
└── 2027/
    └── ...
```

更新索引时使用相对路径：`[报告名称](2026/project-review-2026-03-29.md)`
