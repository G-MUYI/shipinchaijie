## 报告类型说明

### 1. 项目审查报告
- **目的：** 全面评估项目质量，发现问题和改进空间
- **频率：** 重大版本发布前，或定期审查（如每季度）
- **内容：** 架构设计、功能完整性、文档质量、代码质量、测试覆盖、用户体验等
- **命名规范：** `project-review-YYYY-MM-DD.md`

### 2. 评估测试报告
- **目的：** 验证技能在实际测试用例上的表现
- **频率：** 每次技能改进后
- **内容：** 测试用例执行结果、断言通过率、人工审查反馈、改进建议
- **命名规范：** `eval-report-iteration-N-YYYY-MM-DD.md`

### 3. 性能测试报告
- **目的：** 评估技能的性能表现（处理时间、资源消耗等）
- **频率：** 性能优化前后，或定期基准测试
- **内容：** 不同场景的处理时间、API调用次数、内存使用等
- **命名规范：** `performance-report-YYYY-MM-DD.md`

### 4. 描述优化报告
- **目的：** 优化技能描述以提升触发准确率
- **频率：** 发现触发问题时，或定期优化
- **内容：** 触发评估查询、优化迭代过程、最佳描述、触发准确率对比
- **命名规范：** `description-optimization-YYYY-MM-DD.md`
