## 报告类型说明

### 1. 项目审查报告
- **目的：** 全面评估项目质量，发现问题和改进空间
- **频率：** 重大版本发布前，或定期审查（如每季度）
- **命名规范：** `project-review-YYYY-MM-DD.md`

### 2. 评估测试报告
- **目的：** 验证技能在实际测试用例上的表现
- **频率：** 每次技能改进后
- **命名规范：** `eval-report-iteration-N-YYYY-MM-DD.md`

### 3. 性能测试报告
- **目的：** 评估技能的性能表现（处理时间、资源消耗等）
- **频率：** 性能优化前后，或定期基准测试
- **命名规范：** `performance-report-YYYY-MM-DD.md`

## 如何添加新报告
