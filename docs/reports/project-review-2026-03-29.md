**审查日期:** 2026-03-29
**审查工具:** skill-creator

## 📋 项目概览

这是一个高质量的 Claude Code Skill 项目，用于将视频内容拆解为详细的镜头语言描述，并生成可用于 AI 视频生成工具（Sora、可灵、Runway、Seedance 2.0 等）的专业提示词。

**核心技能:** `shipinchaijie` (video-breakdown)

---

## ✅ 优点与亮点

### 1. 架构设计优秀

- **清晰的项目结构** — 代码、文档、模板、参考资料分离明确

## 🔍 发现的问题与改进建议

### 问题 1: 技能描述可能触发不足

**当前描述（SKILL.md frontmatter）:**
```yaml
description: |
  视频拆解提示词生成器 — 将视频拆解为极其细致的镜头语言描述...

  **自动触发条件（满足任一即触发）：**
  1. 用户提供任何视频链接...
  2. 用户提到：视频拆解、视频分析...
  ...
```

**问题:** 描述虽然详细，但可能不够"pushy"（根据 skill-creator 指南，Claude 倾向于"undertrigger"技能）

**建议:** 在描述开头增加更强的触发提示：
```yaml
description: |
  视频拆解提示词生成器 — 当用户提到任何与视频相关的需求时，立即使用此技能。
  将视频拆解为极其细致的镜头语言描述，生成可直接用于 AI 视频生成工具的专业提示词。

  **强制触发场景（只要出现以下任一情况，必须使用此技能）：**
  1. 用户提供任何视频链接（TikTok、抖音、YouTube、B站、快手等平台URL）
  2. 用户提到：视频、拆解、分析、镜头、分镜、运镜、复刻、场景、提示词生成
  3. 用户提到AI视频生成平台：Sora、可灵、Runway、Seedance、即梦、Luma、Pika、Kling
  4. 用户说：帮我分析、我想做视频、这个视频怎么拍的、给我写个提示词
  5. 用户提供本地视频文件路径（.mp4、.mov、.avi等）
  6. 用户描述视频内容并希望生成提示词
  ...
```

### 问题 2: 缺少 workspace 目录

**观察:** 项目根目录没有 `shipinchaijie-workspace/` 目录（README.md 中提到但不存在）

**影响:** 如果要运行评估测试，需要手动创建工作区

**建议:**
- 在项目根目录创建 `shipinchaijie-workspace/` 目录
- 添加 `.gitkeep` 文件保持目录结构
- 或在 README.md 中说明工作区会自动创建

### 问题 3: 评估测试未运行

**观察:** `evals/evals.json` 定义了 5 个测试用例，但没有看到测试结果

**建议:** 运行一轮完整的评估测试：
1. 创建 `shipinchaijie-workspace/iteration-1/` 目录
2. 为每个测试用例生成 with_skill 和 without_skill 的对比结果
3. 使用 skill-creator 的 `generate_review.py` 生成评审界面
4. 根据测试结果优化技能

### 问题 4: 依赖安装脚本可能需要测试

**观察:** 有 `setup.sh` 和 `setup.ps1` 脚本，但未验证是否能正常运行

**建议:**
- 在干净环境中测试安装脚本
- 确保 yt-dlp 安装成功
- 确保 google-genai 库版本兼容

### 问题 5: 输出目录未在 Git 中跟踪

**观察:** `shipinchaijie/output/` 目录存在但被 .gitignore 忽略

**状态:** 这是正确的做法（输出文件不应提交），但可以添加示例输出

**建议:**
- 在 `docs/examples/` 目录中添加 1-2 个示例输出文件
- 展示基础版和专业版的提示词格式
- 展示 Seedance 2.0 格式的提示词

### 问题 6: 手部渐进规则的强调可以更突出

**观察:** 手部渐进规则是这个技能的核心创新点，但在 SKILL.md 中只在"核心原则"部分提到

**建议:**
- 在 Step 8（生成提示词）中再次强调手部渐进规则
- 在每个时间段生成时，自动检查是否符合渐进规则
- 在 validate-output.py 中添加手部渐进规则的自动检测

  --eval-set trigger-eval.json \

  --max-iterations 5
