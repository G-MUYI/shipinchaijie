# video-breakdown Skill v2.0.0 改进总结

## 改进概述

根据用户反馈，对 video-breakdown 技能进行了全面升级，解决了4个核心问题，并新增了多项功能。

## 改进内容

### 1. 添加用户意图询问环节 ✅

**问题：** 之前没有询问用户是否需要复刻重新创作，直接进入视频拆解流程。

**解决方案：**
- 在 Step 1 添加用户意图询问
- 提供三个选项：
  - A) 拆解现有视频 — 分析已有视频的镜头语言和结构
  - B) 复刻重新创作 — 基于现有视频的结构，创作全新内容
  - C) 从零创作 — 直接描述想要的视频内容，生成提示词

**影响：**
- 选择 A 或 B：继续询问视频来源（链接/本地文件/文字描述）
- 选择 C：跳过视频分析，直接进入创意收集环节

**文件修改：**
- `SKILL.md` - Step 1 部分

---

### 2. 生成首帧AI绘图提示词（支持垫图）✅

**问题：** 生成的提示词没有遵循首帧的AI绘图提示词，用户需要垫图功能。

**解决方案：**
- 在 Step 7 添加"生成首帧AI绘图提示词"环节
- 在生成视频提示词之前，必须先生成首帧绘图提示词

**首帧提示词包含：**
1. 主体描述 — 角色/物体的详细外观（服装、发型、表情、姿态）
2. 环境描述 — 场景、背景、空间关系
3. 光线描述 — 光源方向、光线质感、色温
4. 视角描述 — 根据用户选择的视角类型描述镜头角度
5. 画面构图 — 主体在画面中的位置、景别
6. 风格关键词 — 电影实拍质感、照片级写实、摄影机型号和镜头参数

**输出：**
- 独立文件：`first-frame-prompt-{timestamp}.md`
- 告诉用户：
  - 建议先用AI绘图工具生成首帧图片
  - 在视频生成时上传该图片作为垫图
  - 可以大幅提升角色一致性和画面质量

**文件修改：**
- `SKILL.md` - 新增 Step 7
- `SKILL.md` - 核心原则部分

---

### 3. 添加视角选择功能 ✅

**问题：** 用户需要能选择视角，是第一人称POV视角，还是第三人称上帝视角。

**解决方案：**
- 在 Step 2 添加视角选择询问
- 提供三个选项：
  - A) 第一人称POV视角 — 从主角眼睛看出去的视角，观众即主角
  - B) 第三人称上帝视角 — 从外部观察主角的视角
  - C) 混合视角 — 根据场景需要切换视角

**视角对提示词的影响：**

**第一人称POV：**
- 强调"镜头即眼睛"
- 强调"手臂伸入画面"
- 强调"视线跟随"
- 强调"头部晃动"
- 奔跑时镜头上下轻微晃动
- 转头时镜头水平旋转
- 受击时镜头剧烈晃动

**第三人称：**
- 强调"镜头跟随角色"
- 强调"展示全身动作"
- 强调"环绕拍摄"
- 强调"角色在画面中的位置"
- 使用全景或中景展示动作
- 关键动作切换到特写
- 使用环绕镜头展示360度动作

**混合视角：**
- 在每个时间段明确标注当前视角类型

**文件修改：**
- `SKILL.md` - 新增 Step 2
- `SKILL.md` - 核心原则部分
- `SKILL.md` - Step 8 动作描述要求部分

### 4. 优化画面编排、运镜、战斗动作的连贯性和物理逻辑 ✅

**问题：** 需要编排的画面、运镜、战斗动作丝滑连贯，符合物理特性和逻辑性。

**解决方案：**
在 `references/prompt-writing-standards.md` 中新增"动作连贯性与物理逻辑"章节，包含：

#### 4.1 动作分解原则
- 每个动作必须分解为完整的动作链：准备→执行→恢复
- 提供完整示例（跳跃动作的7个阶段）
- 动作链完整性检查清单

#### 4.2 物理规律遵守
- **重力与惯性：** 跳跃弧线、奔跑惯性、抛物线轨迹、落地缓冲
- **力的传递：** 拳击、挥剑、投掷的力量传递路径
- **动量守恒：** 惯性前倾、旋转启动和停止、重物落地震动

#### 4.3 战斗动作编排规则
- **攻击动作三要素：** 蓄力阶段、爆发阶段、收招阶段
- **防御与闪避：** 格挡和闪避的完整动作链
- **连招设计：** 每个动作的结束姿态是下一个动作的起始姿态

#### 4.4 运镜与动作的配合
- **跟随运镜：** 镜头与角色同步移动
- **强调运镜：** 重击推近、闪避侧移、蓄力推近
- **第一人称POV特殊规则：** 晃动、旋转、受击效果
- **第三人称特殊规则：** 景别切换、环绕镜头

#### 4.5 时间感与节奏控制
- **慢动作使用规则：** 仅在关键瞬间使用
- **快节奏动作：** 0.5秒甚至0.3秒细分
- **节奏变化：** 战斗前中后的节奏差异

#### 4.6 环境交互与反馈
- **角色与环境的物理交互：** 脚步、碰撞、风吹、阴影
- **特效与环境的交互：** 火焰照亮、冰霜雾气、雷电闪光、能量波推动

#### 4.7 连贯性检查清单
- 动作连贯性检查（4项）
- 物理逻辑检查（4项）
- 战斗动作检查（4项）
- 运镜配合检查（4项）
- 环境交互检查（4项）

#### 4.8 常见错误与修正
- 错误1：动作跳跃
- 错误2：忽略惯性
- 错误3：力量传递不合理
- 错误4：战斗动作僵硬

## 测试建议

### 测试场景1：从零创作战斗场景
**输入：**
```
我想创作一个15秒的视频：一位火系法师在古老神殿中施展火焰魔法，从站立准备到释放火球击中目标并爆炸。
```

**验证要点：**
- 是否询问用户意图（选择"从零创作"）
- 是否询问视角（选择"第三人称"）
- 是否生成首帧AI绘图提示词
- 动作是否包含完整的准备→执行→恢复过程
- 是否符合物理规律（重力、惯性、力的传递）

### 测试场景2：复刻重新创作
**输入：**
```
我有一个视频链接：[某个战斗视频]，我想保留它的运镜和节奏，但把角色换成水系法师，场景换成海边。
```

**验证要点：**
- 是否询问用户意图（选择"复刻重新创作"）
- 是否询问视角
- 是否生成首帧AI绘图提示词
- 是否保留原视频的运镜结构
- 角色和场景是否完全替换

### 测试场景3：第一人称POV视角
**输入：**
```
我想创作一个第一人称视角的跑酷视频，从屋顶跳跃到另一个屋顶。
```

**验证要点：**
- 是否询问视角（选择"第一人称POV"）
- 运镜描述是否包含"镜头即眼睛"、"手臂伸入画面"
- 跳跃动作是否包含完整的动作链
- 落地是否有缓冲动作
