# video-breakdown v2.0.0 测试用例

## 测试目标
验证新增的4个功能是否正常工作：
1. 用户意图询问（拆解/复刻/从零创作）
2. 首帧AI绘图提示词生成
3. 视角选择（第一人称POV/第三人称/混合）
4. 动作连贯性与物理逻辑优化

## 测试场景

### 场景描述
一个15秒的战斗场景视频：
- 一位火系法师在古老神殿中施展火焰魔法
- 从站立准备→蓄力→释放火球→火球击中目标→爆炸
- 需要展示完整的动作链和物理效果

### 预期工作流程

1. **Step 1: 用户意图询问**
   - 系统应询问：拆解现有视频 / 复刻重新创作 / 从零创作
   - 测试选择：从零创作

2. **Step 2: 视角选择**
   - 系统应询问：第一人称POV / 第三人称上帝视角 / 混合视角
   - 测试选择：第三人称上帝视角

3. **Step 3: 平台选择**
   - 系统应询问：通用平台 / Seedance 2.0
   - 测试选择：通用平台

4. **Step 4: 模板选择**
   - 系统应询问：基础版 / 专业版
   - 测试选择：专业版

5. **Step 5: 首帧AI绘图提示词生成**
   - 系统应生成独立的首帧提示词文件
   - 包含：主体描述、环境描述、光线描述、视角描述、画面构图、风格关键词
   - 文件名：`first-frame-prompt-{timestamp}.md`

6. **Step 6: 视频提示词生成**
   - 总述区应包含：
     - 核心主题
     - 角色设定（详细外观）
     - 环境描述
     - 视觉风格与色调
     - 光线与质感
     - 特效概述
     - 镜头运动（第三人称视角特征）
     - 音频设计

   - 时间线区应包含至少7-8个时间段：
     - 0-2秒：站立准备（手部自然状态）
     - 2-3秒：双脚分开，身体重心下沉，双手开始抬起（蓄力准备）
     - 3-4秒：双手握紧法杖，法杖顶端开始发光（触发瞬间）
     - 4-5秒：火焰能量从法杖向手部蔓延（能量传递）
     - 5-6秒：右手向后拉，身体右转蓄力（攻击准备）
     - 6-7秒：右手向前推出，火球从手掌脱离（释放动作）
     - 7-9秒：火球飞行，留下火焰轨迹（飞行过程）
     - 9-10秒：火球击中目标，产生爆炸（击中瞬间）
     - 10-12秒：爆炸扩散，冲击波推动周围尘土（物理反馈）
     - 12-15秒：法师收回手臂，身体恢复站立姿态（恢复动作）

## 验证要点

### 1. 用户意图询问
- [ ] 是否在开始时询问用户意图？
- [ ] 是否提供了三个选项（拆解/复刻/从零创作）？
- [ ] 选择"从零创作"后是否跳过视频分析？

### 2. 首帧AI绘图提示词
- [ ] 是否生成了独立的首帧提示词文件？
- [ ] 是否包含主体、环境、光线、视角、构图、风格关键词？
- [ ] 是否提到"用于垫图"和推荐的AI绘图工具？
- [ ] 是否强调"电影实拍质感"而非"CG动画"？

### 3. 视角选择
- [ ] 是否询问视角类型？
- [ ] 是否提供了三个选项（第一人称POV/第三人称/混合）？
- [ ] 第三人称视角的描述是否包含"镜头跟随角色"、"展示全身动作"等特征？
- [ ] 运镜描述是否符合第三人称视角（如"镜头环绕"、"从侧面拍摄"）？

### 4. 动作连贯性与物理逻辑
- [ ] 每个动作是否包含完整的准备→执行→恢复过程？
- [ ] 蓄力动作是否描述了重心下沉、肌肉紧绷？
- [ ] 释放动作是否描述了力量传递路径（腿→腰→肩→手）？
- [ ] 火球飞行是否符合抛物线轨迹？
- [ ] 爆炸是否描述了冲击波、尘土飞扬等物理反馈？
- [ ] 相邻时间段的动作是否自然衔接？
- [ ] 是否避免了"瞬移"式的姿态突变？

### 5. 手部描述渐进规则
- [ ] 第1段（0-2秒）是否只描述自然手部外观？
- [ ] 第2-3段（2-5秒）是否描述触发和能量蔓延过程？
- [ ] 第4段之后（5秒+）才描述完整特效状态？

### 6. 文件输出
- [ ] 是否生成了首帧提示词文件？
- [ ] 是否生成了视频提示词文件？
- [ ] 文件命名是否清晰？
- [ ] 是否告诉用户文件保存位置？
