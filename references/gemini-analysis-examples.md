# Gemini 视频分析示例

本文档提供好的分析示例和错误示例对比，帮助 Gemini 理解正确的分析方式。

## 示例1：分段粒度对比

### ❌ 错误示例 - 分段过粗

```json
{
  "segments": [
    {
      "time_range": "00:00.000-00:05.000",
      "scene": "主角站在神殿中",
      "character_action": "准备释放魔法"
    },
    {
      "time_range": "00:05.000-00:12.000",
      "scene": "主角释放魔法攻击敌人",
      "character_action": "施法并击败敌人"
    }
  ]
}
```

**问题：**
- 第一段5秒，第二段7秒，严重超标
- 15秒视频只有2个segments，应该至少8-15个
- 画面描述过于简单，不到20字

- 包含背景微动态（符文闪烁、光芒变化）
- 特效描述精确到触发瞬间

## 示例2：手部描述渐进规则对比

### ❌ 错误示例 - 开头就有特效

```json
{
  "segments": [
    {
      "time_range": "00:00.000-00:02.000",
      "scene": "女巫伸出手",
      "character_action": "手上闪烁着紫色魔法光芒，指尖环绕着能量",
      "effects": "手部被紫色能量包裹"
    }
  ]
}
```

**问题：**
- 开头就描述手部有特效，违反渐进规则
- 没有自然状态、没有触发过程
- AI无法理解特效是如何出现的

## 示例3：画面描述详细度对比


- scene描述只有7个字，远低于50字要求

## 示例5：背景微动态对比

### ❌ 错误示例 - 缺少背景微动态

```json
{
  "segments": [
    {
      "time_range": "00:00.000-00:01.500",
      "scene": "主角站在森林中，周围是树木和草地",
      "character_action": "静止站立"
    }
  ]
}
```

**问题：**
- 背景描述是静态的，没有任何运动元素
- 会导致AI生成"死水"般的静止背景
- 缺少生命力和真实感

### ✅ 正确示例 - 包含背景微动态

```json
{
  "segments": [
    {
      "time_range": "00:00.000-00:01.500",
      "scene": "主角站在茂密森林的空地中央，周围是高大的橡树（树干直径约1米），树叶在微风中轻轻晃动，阳光透过树叶间隙洒下斑驳的光影，光斑随着树叶摆动在地面上缓慢移动。地面是覆盖着青苔的土地，草地上的野花随风轻轻摇曳。远处可见树林深处有薄雾缓慢飘动，几只小鸟从树枝间飞过。空气中飘浮着细小的花粉和灰尘微粒，在阳光中清晰可见。",
      "character_action": "静止站立，呼吸时胸部微微起伏"
    }
  ]
}
```

**优点：**
- 包含多个背景微动态元素：
  - 树叶晃动
  - 光斑移动
  - 野花摇曳
  - 薄雾飘动
  - 小鸟飞过
  - 花粉/灰尘飘浮
- 背景有生命力，不是静止画面
- 符合真实世界的物理规律

## 总结：Gemini 分析的关键要点

1. **分段必须极细**：单段不超过2秒，segments数量 ≥ 时长÷2
2. **画面描述必须详细**：每段至少50字，包含角色、场景、光线、构图、微动态
3. **手部描述必须渐进**：严格遵循五阶段（自然→手势→捏爆→蔓延→持续）
4. **战斗必须完整**：包含完整的7阶段Boss战，不可跳过
5. **背景必须有微动态**：每段至少一个运动元素，防止静止背景

请在分析视频时严格遵循这些规则。
