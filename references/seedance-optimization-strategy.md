# Seedance 字数限制与画面细节兼容策略

## 问题分析

**核心矛盾：**
- Seedance 要求 100-260 字提示词
- 详细画面描述通常需要 400+ 字
- 直接压缩会导致关键细节丢失

## 解决方案：三层分工策略

### 策略 1：@素材承载细节（最重要）

**原则：能用图说的，不用文字写**

| 细节类型 | 传统方式（文字） | Seedance 方式（素材） | 节省字数 |
|---------|----------------|---------------------|---------|
| 角色外观 | 50-80字描述 | @图片1 锁脸 | 节省 70字 |
| 场景环境 | 40-60字描述 | @图片2 场景参考 | 节省 50字 |
| 手部细节 | 30-50字描述 | @图片1 首帧（含手部） | 节省 40字 |
| 光影质感 | 40-60字描述 | 首帧图自带光影 | 节省 50字 |

- 运镜、画面、光线、人物动作、手部状态、情绪、特效、台词、声音、转场

## 完整工作流程

### 步骤 1：生成详细拆解报告（不受字数限制）

正常生成完整的拆解报告，包含所有细节：
- 基础信息、视觉风格、光线质感、运镜、主角、手部、场景、动作、音频等
- 这部分用于理解视频，字数不限

### 步骤 2：提取首帧绘图提示词（单独生成）

从拆解报告中提取关键视觉元素，生成首帧图的绘图提示词：
- 角色外观（发型、服装、肤色、指甲）
- 手部状态（自然状态 + 魔法球）
- 场景环境（背景、道具、氛围）
- 光线质感（光源、色调、材质）

**这部分不计入视频提示词字数！**

### 步骤 3：生成精简视频提示词（100-260字）

按照三层分工策略生成：

```markdown
@图片1为首帧。[视角]，[场景一句话]。

[时间1]：[运镜]，[核心动作]，[关键特效]。
[时间2]：[运镜]，[核心动作]，[关键特效]。
[时间3]：[运镜]，[核心动作]，[关键特效]。

[摄影机型号]，[画质]。
禁止出现文字、字幕、LOGO或水印。
```

## 关键细节保留检查清单

### ✅ 必须保留的信息

- [ ] 视角类型（第一人称POV/第三人称/俯视等）

## 常见问题解答

### Q1: 首帧图能承载多少细节？

**A:** 一张高质量的首帧图可以包含：
- 角色完整外观（发型、服装、配饰、肤色）
- 手部自然状态（指甲、姿态、魔法球）
- 场景环境（背景、道具、空间层次）
- 光影氛围（光源、色调、质感）
- 视角和构图（POV、景别、角度）

**相当于节省 150-200 字的文字描述。**

### Q2: 如果视频有多个场景怎么办？

**A:** 使用多张参考图：
```
@图片1为首帧（地下神殿场景）。
@图片2为第二场景（森林空地）。

0-5秒：[地下神殿动作]
5-10秒：场景切换到@图片2，[森林空地动作]
```

每个场景一张图，文字只写动作和转场。

### Q3: 特效细节会丢失吗？

**A:** 不会。特效的关键信息保留在文字中：
- **触发时机**："捏爆魔法球瞬间"
- **蔓延路径**："从手指→手臂→全身"
- **视觉效果**："蓝色能量"、"火焰冲击波"
- **持续状态**："能量持续环绕双手"

首帧图只需展示特效触发前的自然状态。

### Q4: 运镜复杂度会受限吗？

**A:** 遵循 Seedance 的"单运镜原则"：
- 每段只写一种运镜（推近/震动/跟随）
- 复杂运镜用 @视频1 参考
- 文字只写运镜类型，不写轨迹细节

### Q5: 如何确保字数在 100-260 范围内？

**A:** 使用字数分配公式：

```
开头（30字）+ 时间线（每段25字 × 4-5段）+ 结尾（20字）
= 30 + 100-125 + 20 = 150-175字（最佳范围）
```

如果超标，优先删除：
1. 光线描述（首帧图已包含）
2. 情绪表情（首帧图已包含）
3. 冗余修饰词（"缓慢地"→"缓慢"）
4. 重复信息（每段不重复写场景）

## 总结

**核心原则：能用图说的，不用文字写。**

| 传统方式 | Seedance 方式 | 效果 |
|---------|--------------|------|
| 400+ 字全文字描述 | 150-200 字 + 首帧图 | 字数减少 50%，细节不丢失 |
| 每段 10 个字段 | 每段 3 个核心要素 | 聚焦关键信息 |
| 重复描述光影氛围 | 首帧图一次性承载 | 避免冗余 |
| 详细角色外观描述 | @图片1 锁脸 | 一致性更好 |

**这不是妥协，而是更聪明的分工策略。**

文字负责：动作、节奏、特效变化
图片负责：外观、场景、光影、质感

两者结合，既满足 Seedance 的字数限制，又保留了所有画面细节。
