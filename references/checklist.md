# 提示词生成检查清单

本文档汇总所有检查清单，按生成阶段分类。生成提示词时对照检查，确保质量。

---

## 生成前检查

### 1. 手部描述五阶段检查

**适用场景**：包含手部特效的视频（魔法、能量、变身等）

- [ ] 第1阶段（自然）：手部是否只描述了自然外观和魔法球？
- [ ] 第2阶段（手势）：是否描述了星座专属手势或结印动作？
- [ ] 第3阶段（捏爆）：是否详细描述了捏爆魔法球的瞬间？
- [ ] 第4阶段（爆发蔓延）：是否详细描述了特效从爆裂点向手部蔓延的过程？
- [ ] 第5阶段（持续）：后续每一段是否都明确描述了特效的持续存在？
- [ ] 整体：是否能清晰看出"自然→手势→捏爆→爆发蔓延→持续"的五阶段过渡？

**参考文档**：`prompt-writing-standards.md` 第5-150行

---

### 2. 平台选择检查

- [ ] 是否明确了目标平台（通用平台 / Seedance 2.0）？
- [ ] 如果是Seedance，是否选择了对应的内容类型骨架？
- [ ] 是否读取了对应平台的输出骨架文件？

---

### 3. 视角选择检查

- [ ] 是否明确了视角类型（第一人称POV / 第三人称 / 混合）？
- [ ] 提示词描述是否符合所选视角的特点？
  - 第一人称：强调"镜头即眼睛"、"手臂伸入画面"
  - 第三人称：强调"展示全身动作"、"角色在画面中的位置"

---

## 生成中检查

### 4. 动作连贯性检查

- [ ] 每个动作是否有完整的准备→执行→恢复过程？
- [ ] 相邻时间段的动作是否自然衔接？
- [ ] 是否有"瞬移"式的姿态突变？
- [ ] 连续动作的重心转移是否合理？

**参考文档**：`prompt-writing-standards.md` 第435-699行

---

### 5. 物理逻辑检查

- [ ] 跳跃/落地是否符合重力规律？
- [ ] 快速移动/停止是否考虑惯性？
- [ ] 力量传递路径是否正确（腿→腰→肩→手）？
- [ ] 物体运动轨迹是否符合物理规律？

---

### 6. 背景微动态检查

- [ ] 每段画面描述中是否包含至少一个背景微动态元素？
- [ ] 微动态是否与场景类型匹配？
- [ ] 微动态是否细微、持续，不喧宾夺主？

**8种场景类型参考**：
- 自然户外：树叶/草丛晃动
- 城市街道：车流/灯光闪烁
- 室内场景：窗帘/烛光/灰尘
- 水边场景：水面涟漪/倒影扭曲
- 天空/高处：云层飘移/鸟群
- 夜景：霓虹闪烁/星光
- 神殿/洞窟：火把跳动/符文明暗
- 雨雪天气：雨丝/雪花飘落

**参考文档**：`prompt-writing-standards.md` 第584-615行

---

### 7. 材质交互检查

- [ ] 是否描述了物体的重量感（而非仅说"金属"、"木头"）？
- [ ] 运动中的物体是否符合其材质的物理特性（刚性/柔性）？
- [ ] 物体表面是否有正确的环境反射/折射描述？
- [ ] 能量/特效是否有光照衰减，是否照亮周围环境？
- [ ] 布料/头发等柔性物体是否有符合物理的飘动延迟？

**参考文档**：`prompt-writing-standards.md` 第331-428行

---

### 8. 真实感渲染词汇检查

**禁用词汇**（必须避免）：
- [ ] 是否使用了"CG动画"、"3D渲染"、"CG特效"？
- [ ] 是否使用了"动画风格"、"卡通化"、"渲染感"？

**推荐词汇**（必须包含）：
- [ ] 是否包含"电影级写实摄影"、"真实拍摄质感"？
- [ ] 是否包含"次表面散射"、"全局光照"、"体积光"？
- [ ] 是否包含摄影机型号和镜头参数？

**参考文档**：`prompt-writing-standards.md` 第302-330行

---

## 生成后检查（通用平台）

### 9. 结构完整性检查

- [ ] 是否包含首帧AI绘图提示词？
- [ ] 总述区是否包含所有必需字段（核心主题/角色设定/环境/视觉风格/光线/特效/镜头/音频）？
- [ ] 时间线区每段是否包含所有必需字段（运镜/画面/光线/动作/手部/情绪/特效/台词/声音/转场）？
- [ ] 是否包含禁止项声明？

---

### 10. 时间分段检查

- [ ] 常规动作是否按1-2秒/段分段？
- [ ] 关键动作是否按0.5秒/段分段？
- [ ] 高潮节点是否按0.3秒/段分段？
- [ ] 段落数是否 ≥ 总时长÷2？

---

### 11. 战斗场景专项检查

**适用场景**：包含战斗/对抗的视频

- [ ] 攻击是否有蓄力→爆发→收招三阶段？
- [ ] 连招之间是否有合理的衔接动作？
- [ ] 防御/闪避动作是否完整？
- [ ] 受击反应是否符合力的方向？
- [ ] 如有Boss战，是否包含完整7阶段？

**参考文档**：`combat-choreography-guide.md`

---

## 生成后检查（Seedance 2.0）

### 12. Seedance四大原则检查

- [ ] **原则1：字数控制** - 字数是否符合场景复杂度？
  - 简单场景：100-300字
  - 标准场景：300-600字
  - 复杂场景：600-1000字

- [ ] **原则2：单运镜** - 每个时间段是否只用了一种运镜？

- [ ] **原则3：禁止文学化** - 是否避免了隐喻/诗意描述？
  - ❌ "时光在指尖流淌"
  - ✅ "女性手指翻动旧照片，照片泛黄卷边"

- [ ] **原则4：素材优先** - 角色外观是否用@图片引用，而不是大量文字描述？

**参考文档**：`seedance-guide.md` 第12-148行

---

### 13. Seedance内容类型匹配检查

- [ ] 是否选择了与内容类型匹配的骨架？
  - 动作/打斗 → 时间轴分镜式
  - 舞蹈/MV → FORMAT声明式或时间轴式
  - 婚礼/情感 → 时间轴分镜式
  - 动物拟人 → 步骤分镜式
  - 恐怖/悬疑 → 时间轴分镜式
  - 科幻/赛博朋克 → FORMAT声明式
  - 简单/趣味 → 自然叙述式

**参考文档**：`seedance-guide.md` 第399-451行

---

### 14. Seedance音效描述检查

- [ ] 是否在末尾加入了音效描述？
- [ ] 音效是否按三层结构描述？
  - 音乐层
  - 环境音层
  - 动作音层

**参考文档**：`seedance-guide.md` 第453-487行

---

### 15. Seedance图片生成提示词检查

- [ ] 所有@图片的生成提示词是否使用中文？
- [ ] 图片提示词是否与视频主题风格匹配？
- [ ] 是否避免了"CG动画"、"3D渲染"等词汇？

---

## 多主题视频专项检查

### 16. 多主题拆分检查

**适用场景**：视频包含多个独立主题（不同角色/场景/故事线）

- [ ] 是否为每个主题生成了独立的完整文件？
- [ ] 每个文件是否有完整的总述区和时间线区？
- [ ] 每个文件的角色描述是否与该主题完全匹配？
- [ ] 文件命名是否清晰标注了主题？
- [ ] 是否告诉用户生成了几个文件，每个对应哪个主题？

**参考文档**：`prompt-writing-standards.md` 第836-971行

## 易犯错误对照表

生成后对照以下表格，出现任何一条都要纠正：

| 问题 | 正确做法 |
|------|---------|
| 手部开头就有特效（"手上闪烁着光芒"） | 开头只描述自然手部，特效从触发瞬间才出现 |
| 特效触发后消失、不持续 | 特效一旦触发，后续每段都必须描述持续状态 |
| 背景是静止死板的 | 每段必须有至少一个背景微动态元素 |
| 战斗场景跳过Boss战阶段 | 必须包含7阶段完整Boss战 |
| 一镜到底直接硬切场景 | 场景切换必须有物理遮挡物 |
| Seedance提示词字数不合理 | 根据复杂度控制（简单100-300，标准300-600，复杂600-1000） |
| Seedance一段中混写多种运镜 | 每段只用一种运镜 |
| 使用"CG动画""3D渲染"等词汇 | 只用"电影级写实摄影""真实拍摄质感" |
| 画面描述只写动作不写场景 | 每段必须描述具体画面内容（谁、在哪、背景是什么）|
| Seedance提示词缺少音效描述 | 末尾必须加音效列表（音乐层+环境音层+动作音层）|
| Seedance提示词使用文学化描述 | 禁止隐喻/诗意，只描述可见画面 |
| Seedance长提示词未使用FORMAT声明 | MV/科幻/多镜头场景应使用FORMAT声明式骨架 |
| 内容类型与骨架不匹配 | 参考seedance-style-library.md选择对应骨架 |

---

## 使用建议

1. **生成前**：先检查第1-3项，确保基础设置正确
2. **生成中**：边写边检查第4-8项，确保质量
3. **生成后**：根据平台检查第9-15项，确保完整性
4. **最后**：对照易犯错误表，逐项排查

**建议将本检查清单打印或保存为快捷参考。**
